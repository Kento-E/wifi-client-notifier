# イシュー自動クローズ機能

このドキュメントでは、Pull Request（PR）がマージされた際にイシューを自動的にクローズする方法について説明します。

## 概要

GitHubのネイティブ機能を使用して、PRがmainブランチにマージされると、関連するイシューを自動的にクローズできます。

## 使い方

### 基本的な使い方

PR説明またはコミットメッセージに以下のキーワードとイシュー番号を含めます：

```markdown
Closes #10
```

これにより、PRがマージされると自動的にイシュー#10がクローズされます。

### 利用可能なキーワード

以下のキーワードが使用できます（すべて英語）：

- **`Closes #番号`** - 一般的なイシューのクローズ
- **`Fixes #番号`** - バグ修正の場合
- **`Resolves #番号`** - 問題解決の場合

### 複数のイシューをクローズ

複数のイシューを一度にクローズする場合：

```markdown
Closes #10, #15, #20
```

または、別々に記述することもできます：

```markdown
Closes #10
Fixes #15
Resolves #20
```

## PR説明の例

```markdown
## 変更概要

WiFi接続監視機能のバグを修正しました。

## 変更理由

ログイン時にタイムアウトが発生する問題を解決するため。

## 主な変更点

- `wifi_notifier.py`: タイムアウト処理を追加

## 関連Issue

Fixes #10

## 動作確認

- [x] ローカルでの動作確認
- [x] タイムアウトケースのテスト
```

## PRテンプレートの使用

新しいPRを作成すると、自動的にテンプレートが表示されます。
`Closes #` の部分にイシュー番号を入力してください。

## 注意事項

1. **キーワードは英語で記述する必要があります**（GitHub仕様）
2. PR説明やコミットメッセージに含める必要があります
3. PRがmainブランチにマージされた時点でイシューがクローズされます
4. PRがクローズされただけではイシューはクローズされません（マージが必要）

## トラブルシューティング

### イシューが自動的にクローズされない場合

以下を確認してください：

- [ ] PRがマージされている（クローズだけでは不十分）
- [ ] キーワード（Closes, Fixes, Resolves）が正しく記述されている
- [ ] イシュー番号が正しい
- [ ] PR説明またはコミットメッセージに含まれている
